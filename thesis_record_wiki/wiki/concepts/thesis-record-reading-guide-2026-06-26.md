# Purpose

Use this page when returning to the KGAS/thesis record after time away. It gives short reading paths through the wiki so the archive stays navigable without flattening the record into one oversimplified story.

# First Rule

The wiki is a navigation and synthesis layer. The raw archive remains the historical source of truth. Do not mutate `archive_full_record/`, publish raw material, or treat wiki summaries as a substitute for source review when precision matters.

# 15-Minute Path

Read these in order:

1. [Private KGAS README Draft 2026 06 26](/wiki/concepts/private-kgas-readme-draft-2026-06-26.md) - fastest current orientation.
2. [Private KGAS README Claim Audit 2026 06 26](/wiki/concepts/private-kgas-readme-claim-audit-2026-06-26.md) - what the README can and cannot claim.
3. [Safe Thesis Organization Closeout 2026 06 26](/wiki/concepts/safe-thesis-organization-closeout-2026-06-26.md) - what is complete, what is gated, and what not to do.

Use this path when the goal is: "What is this work, what is safe to say, and what should I avoid?"

# 60-Minute Path

Read the 15-minute path, then:

1. [Thesis Recovery Current State 2026 06 26](/wiki/concepts/thesis-recovery-current-state-2026-06-26.md) - active checkpoint tying together runtime proof, proposal history, export boundaries, and remaining gates.
2. [KGAS Dissertation Claim Map](/wiki/concepts/kgas-dissertation-claim-map.md) - separates vision, proposal claim, validation design, runtime proof, and governance boundary.
3. [Theory Schema Application Lineage](/wiki/concepts/theory-schema-application-lineage.md) - the core methods story: paper to schema to representation to application to validation.
4. [Uncertainty Framework Consolidation 2026 06 26](/wiki/concepts/uncertainty-framework-consolidation-2026-06-26.md) - why uncertainty should be read as an evolving evidence-discipline lineage.
5. [Public Export Security Boundary](/wiki/concepts/public-export-security-boundary.md) - why public sharing requires a derived, scanned artifact.

Use this path when the goal is: "Could this become a portfolio page, academic note, dissertation restart, or implementation revival?"

# Stop Lines

Stop and get explicit Brian approval before:

- publicizing any repo, export, archive, or derived page;
- reading raw sensitive datasets beyond manifest-level summaries;
- quoting raw dataset text, identifiers, handles, credentials, logs, or local config values;
- executing Neo4j cleanup rather than dry-run/source-scoped planning;
- changing `archive_full_record/`;
- claiming HSPC/IRB approval, dissertation completion, production readiness, or full validation;
- converting private README/narrative drafts into public copy.

# Recommendation

Default to the 15-minute path for orientation and the 60-minute path before any planning. Use the deep review path only when the next step is a serious dissertation, publication, validation, or implementation decision.
