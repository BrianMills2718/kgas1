# Summary

Complexity conservation in theory application is the lesson that a universal framework can remove theory-specific code paths, but it does not remove theory-specific work. That work moves into schema configuration, prompt design, stage validation, quality metrics, and expert review.

The universal applicator critique states this directly: the approach generalizes the multi-stage application pattern, but its effectiveness depends on well-crafted schema configurations. [1]

# Implication

A schema-driven framework is still valuable because schemas are easier to maintain, compare, and modify than scattered custom code. But it is not enough to say "the framework is universal." Each theory still needs:

- explicit application stages
- output schemas per stage
- examples and criteria
- validation checks
- quality metrics
- failure-mode handling
