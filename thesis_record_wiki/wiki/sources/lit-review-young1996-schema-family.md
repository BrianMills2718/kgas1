# Summary

The preserved `schemas/young1996/` folder contains six files: five Young 1996 schema variants and one execution prompt example. Its aggregate hash over sorted filenames and per-file hashes is `3830ddd0e259bbf29fb1c604e62e4b3d5036522381b776f5ba08c7b2d2b9f733`. [1]

This is a mature Young/WorldView schema family rather than a single generated schema.

# Variants

| File | Role |
|---|---|
| `young1996_computational_complete.yml` | complete computational implementation with core elements, algorithms, execution template, and validation checks. [2] |
| `young1996_enhanced.yml` | application-oriented schema with application stages, output mapping, and summary rules. [3] |
| `young1996_multi_pass.yml` | multi-pass extraction artifact with notation system, formal rules, algorithms, implementation, evaluation, and examples. [4] |
| `young1996_simplified.yml` | simplified network schema with nodes, connections, properties, modifiers, notation, and rules. [1] |
| `young1996_theory_only.yml` | theory-only schema preserving mechanisms, formulas, notation, and rules. [1] |
| `execution_prompt_example.txt` | prompt for executing cognitive-mapping algorithms over a Carter speech. [5] |

# Computational Content

`young1996_computational_complete.yml` defines relationship categories, truth values, modifiers, conjunctions, and algorithms for concept extraction, relationship extraction, salience calculation, dependency calculation, connectedness calculation, and directed-walk reasoning. Its validation checks require valid relationship categories, truth values, positive salience scores, structural measures, and evidence snippets. [2]

The execution prompt makes the schema operational: it instructs the model to extract concepts, classify subject-relation-object triples, assign truth values and modifiers, count actual relationship frequencies for salience, compute structural measures, and output JSON over a Carter speech. [5]

# Application Content

`young1996_enhanced.yml` adds universal-applicator-style stages for extraction, filtering, structuring, and analysis. It also maps concepts, relationships, and conjunctions to output fields. [3]

This links directly to [Lit Review Universal Theory Applicator](/wiki/sources/lit-review-universal-theory-applicator.md), where `young1996_enhanced.yml` and `young_universal_test.yml` are used as the concrete example.

# Interpretation

This folder is one of the strongest examples of [Schema Variant Drift](/wiki/concepts/schema-variant-drift.md) becoming productive rather than merely noisy. The variants preserve different operational surfaces:

- theory-only representation
- simplified schema
- multi-pass extraction record
- universal-applicator schema
- complete computational algorithm spec
- execution prompt over real text

Together they show the thesis direction: theory schemas were becoming executable analysis programs, not just ontology descriptions.
