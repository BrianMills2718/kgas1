# Archive Logic

The README separates "working system reality" from "theoretical materials." The working reality is simple: text, embeddings, database, basic chaining, contracts/adapters, Neo4j/SQLite, and KISS. The theoretical side includes complex analysis frameworks, academic proposals, sophisticated automation, uncertainty propagation, and dynamic theory/tool concepts that exceed current implementation needs. [2]

This is not a low-value junk archive. It is a deliberate separation between:

- what was practical enough to keep in the working system, and
- what was intellectually valuable but too complex for the current implementation boundary. [2]

# Preserved Concept Families

The README names four major preserved concept families:

- Analysis philosophy: text-internal vs. text-external analysis, theory-to-analysis mapping, text-analysis dimensions, and methodological failure modes. [2]
- Advanced architecture: cross-modal orchestration, dynamic tool generation from theory schemas, and high-dimensional uncertainty frameworks. [2]
- Research methodology: two-stage analysis, academic validation, real-data validation planning, and analysis recovery strategies. [2]
- Implementation concepts: theory-to-code automation, adaptive tool enhancement, and quantitative theory implementation patterns. [2]

# Interpretation

This directory is central to preserving Brian's thesis evolution because it contains ideas that were important enough to keep but intentionally separated from the working KGAS implementation. It should be read as intellectual lineage and future-option space, not as current system status.

The most important caution is to preserve the archive's own boundary: do not re-import complexity into the working-system narrative just because it is conceptually rich. The value is in knowing what was considered, why it was archived, and when it might become relevant again. [2]
