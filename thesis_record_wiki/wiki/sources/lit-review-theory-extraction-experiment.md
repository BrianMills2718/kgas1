# Summary

`experiments/lit_review` is a separate thesis experiment subsystem inside the large Digimons lineage bundle. Its goal is automated theory extraction and application: academic paper -> schema extraction -> data application -> comparable results. [1][2]

The subsystem is 51M with 2,768 files and its own `.git` directory. This first slice ingests only overview, docs, validation summary, and major phase summaries.

# Core Method

The project uses a three-phase processing pipeline:

1. comprehensive vocabulary extraction
2. ontological classification
3. theory-adaptive schema generation [1][2]

Supported representations include property graphs, hypergraphs, table/matrix structures, sequences, trees, timelines, statistical/logical/causal models, and hybrids. [2]

# Schema-Based Ontologies Line

The docs README frames the system as a schema-based ontology and literature analysis pipeline with 50+ YAML schemas, an interactive web UI, framework-specific visualizations, and theoretical domains such as argumentation, behavior change, information disorder, cultural evolution, persuasion, social marketing, operational code, and decision theory. [3]

# Validation Claims

The validation summary reports methodology robustness testing across academic papers, cross-domain validation, complexity testing, and architecture optimization. It claims 100% validation completion and 100% model-type accuracy after experimental testing. [4]

Phase summaries then claim:

- Phase 1: balanced purpose classification across descriptive, explanatory, predictive, causal, and intervention purposes. [5]
- Phase 4: balanced integration pipeline completed with 100/100 score. [6]
- Phase 5: cross-purpose reasoning engine completed with perfect balance score. [7]
- Phase 6: production validation certified with 0.910 overall score. [8]

# Caution

These phase summaries are internal evidence claims from the subsystem. They should be preserved, but not treated as externally validated thesis success without a later review that applies [Evidence Claim Discipline](/wiki/concepts/evidence-claim-discipline.md).
